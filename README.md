# WhatToEat 🍽️

> **You already know what you want.**
>
> **You just don't know it yet.**

## What is WhatToEat?

WhatToEat helps people discover what they actually want to eat.

Instead of overwhelming users with endless lists of restaurants or dishes, WhatToEat guides them through a short decision process and recommends the dish that best matches what they're craving right now.

---

## The Problem

Every day, millions of people spend time trying to decide what to eat.

The problem isn't a lack of options.

The problem is having too many.

Food delivery apps assume users already know what they want.

In reality, many people only have a vague feeling:

- Something spicy.
- Something comforting.
- Something to share.
- Anything, as long as I don't have to think too much.

---

## Our Solution

WhatToEat asks the right questions to uncover what the user is actually craving.

Instead of showing hundreds of options, it recommends one dish with the highest match.

If it misses, the user can instantly ask for another recommendation based on the same preferences.

---

## MVP

The first version focuses on one goal:

**Recommend the right dish.**

The MVP intentionally does **not** include:

- Restaurant search
- Food delivery integration
- Maps
- Prices
- Reservations

Those features can be added later.

First, we validate the recommendation engine.

---

## Core Principles

- Help users discover what they want.
- Reduce decision fatigue.
- Recommend first, browse second.
- Learn from user feedback over time.
- Explain recommendations when requested.
